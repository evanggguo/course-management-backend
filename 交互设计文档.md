## 三、用户注册与登录

### 3.1 注册流程

```mermaid
sequenceDiagram
    actor 用户
    participant 前端
    participant 后端
    participant 数据库

    用户->>前端: 填写注册表单
    Note over 前端: 校验: 用户名/密码不为空<br/>密码长度≥6位
    前端->>后端: POST /api/auth/register
    后端->>数据库: 查询用户名是否存在
    数据库-->>后端: 返回结果

    alt 用户名已存在
        后端-->>前端: 400 用户名已被使用
        前端-->>用户: 显示错误提示
    else 用户名可用
        后端->>数据库: 插入用户记录(密码BCrypt加密)
        数据库-->>后端: 成功
        后端-->>前端: 200 注册成功
        前端-->>用户: 提示成功，跳转登录页
    end
```

### 3.2 登录流程

```mermaid
sequenceDiagram
    actor 用户
    participant 前端
    participant 后端

    用户->>前端: 输入用户名+密码
    前端->>后端: POST /api/auth/login
    后端->>后端: 验证密码(BCrypt比对)

    alt 验证失败
        后端-->>前端: 500 用户名或密码错误
        前端-->>用户: 红色错误提示
    else 验证成功
        后端-->>前端: 200 { token, role, username }
        前端->>前端: 存储token到localStorage
        alt role = TEACHER
            前端-->>用户: 跳转 /teacher/courses
        else role = STUDENT
            前端-->>用户: 跳转 /student/courses
        end
    end
```

### 3.3 注册页面字段说明

| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|----------|
| 用户名 | 文本输入 | ✅ | 不为空，唯一 |
| 密码 | 密码输入 | ✅ | 不为空 |
| 角色 | 单选（教师/学生） | ✅ | 必选其一 |
| 真实姓名 | 文本输入 | ❌ | — |
| 邮箱 | 邮箱输入 | ❌ | 格式校验 |

## 四、教师端交互流程

### 4.1 课程管理

```mermaid
flowchart TD
    A[进入课程管理页] --> B[展示课程列表表格]
    B --> C{用户操作}

    C -->|点击新增| D[弹出课程表单Modal]
    D --> E[填写: 课程名/描述/单价]
    E --> F{提交}
    F -->|成功| G[列表刷新，新课程出现]
    F -->|失败| H[Modal内显示错误]

    C -->|点击编辑| I[弹出预填数据的Modal]
    I --> J[修改字段]
    J --> K{提交}
    K -->|成功| L[列表数据更新]

    C -->|点击删除| M[弹出确认对话框]
    M -->|确认| N[删除成功，列表移除]
    M -->|取消| B

    C -->|切换状态| O[ACTIVE ↔ INACTIVE]
```

**课程表单字段：**

| 字段 | 组件 | 说明 |
|------|------|------|
| 课程名称 | Input | 必填 |
| 课程描述 | TextArea | 选填 |
| 每课时单价（元） | InputNumber | 必填，≥0 |
| 状态 | Switch | 默认启用 |

### 4.2 课程安排

```mermaid
flowchart TD
    A[进入课程安排页] --> B[展示排课列表]
    B --> C{操作}

    C -->|新增排课| D[弹出排课Modal]
    D --> E[选择课程下拉]
    E --> F[选择开始时间 DateTimePicker]
    F --> G[选择结束时间 DateTimePicker]
    G --> H{时间校验}
    H -->|开始时间 >= 结束时间| I[提示时间无效]
    H -->|时间合法| J[提交]
    J -->|成功| K[排课出现在列表]

    C -->|修改排课| L[仅限 AVAILABLE 状态]
    C -->|删除排课| M[确认后删除]
```

**排课状态说明：**

| 状态 | 含义 | 可操作 |
|------|------|--------|
| AVAILABLE | 可被学生预约 | 可编辑/删除 |
| BOOKED | 已被学生预约 | 只读 |
| COMPLETED | 已完成 | 只读 |
| CANCELLED | 已取消 | 只读 |

## 五、学生端交互流程

### 5.1 购买课时（核心流程）

```mermaid
sequenceDiagram
    actor 学生
    participant 前端
    participant 后端
    participant DB as 数据库

    学生->>前端: 在课程列表点击"购买课时"
    前端->>前端: 弹出购买Modal
    学生->>前端: 输入购买课时数量
    前端->>前端: 实时计算 = 数量 × 单价
    学生->>前端: 确认购买

    前端->>后端: POST /api/student/courses/{id}/purchase
    后端->>DB: 查询 student_course_hours
    alt 记录已存在
        后端->>DB: UPDATE 累加课时
    else 首次购买
        后端->>DB: INSERT 新记录
    end
    后端->>DB: INSERT hour_purchase_records
    DB-->>后端: 成功
    后端-->>前端: 200 购买成功
    前端-->>学生: 提示成功，关闭Modal
```

### 5.2 预约课程（核心流程）

```mermaid
sequenceDiagram
    actor 学生
    participant 前端
    participant 后端
    participant DB as 数据库

    学生->>前端: 在可预约课表中点击"预约"
    Note over 前端: 按钮仅在该课程有剩余课时时可点击

    前端->>后端: POST /api/student/bookings
    后端->>DB: 查询剩余课时

    alt 剩余课时 = 0
        后端-->>前端: 400 课时不足
        前端-->>学生: 提示购买课时
    else 课时充足
        后端->>DB: UPDATE 课时 -1（used+1, remaining-1）
        后端->>DB: UPDATE schedule状态 → BOOKED
        后端->>DB: INSERT bookings记录
        后端-->>前端: 200 预约成功
        前端-->>学生: 提示成功
    end
```

### 5.3 取消预约（退还课时）

```mermaid
sequenceDiagram
    actor 学生
    participant 前端
    participant 后端
    participant DB as 数据库

    学生->>前端: 在"我的预约"中点击"取消"
    前端->>前端: 弹出确认对话框
    学生->>前端: 确认取消

    前端->>后端: PUT /api/student/bookings/{id}/cancel
    后端->>DB: UPDATE booking状态 → CANCELLED
    后端->>DB: UPDATE schedule状态 → AVAILABLE
    后端->>DB: UPDATE 课时 +1（used-1, remaining+1）
    后端-->>前端: 200 取消成功
    前端-->>学生: 列表刷新，课时余额恢复
```

## 八、错误处理交互

### 8.1 HTTP 错误码对应处理

| 状态码 | 场景 | 前端表现 |
|--------|------|----------|
| 400 | 参数错误（如课时不足） | Modal 内或页面顶部红色提示 |
| 401 | Token 失效/未登录 | 清除本地存储，跳转登录页 |
| 403 | 角色无权限 | 提示"无权访问"，跳转对应首页 |
| 500 | 服务器内部错误 | 提示"操作失败，请稍后重试" |

### 8.2 Token 过期处理流程

```mermaid
flowchart TD
    A[发起 API 请求] --> B[Axios 响应拦截器]
    B --> C{状态码?}
    C -->|401 或 403| D[清除 localStorage]
    D --> E[跳转 /login]
    C -->|其他| F[正常处理响应]
```

### 8.3 表单校验规则汇总

| 场景 | 校验规则 |
|------|----------|
| 注册用户名 | 不为空 |
| 注册密码 | 不为空 |
| 创建课程名称 | 不为空 |
| 课时单价 | 数字，≥ 0 |
| 排课时间 | 结束时间必须晚于开始时间 |
| 购买课时数 | 正整数，≥ 1 |
